CSS POSITIONING:
--------------------
Positioning controls where an element appears on the webpage.
it allows us to move elements from their normal; location.

Syntax:
------
postion:value;

Types of position:
-------------
1. static
2. relative
3. absolute
4. fixed
5. sticky

Position 1: Static:
-------------------
Static is the default position of every HTML element.
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        .box{
            width: 100px;
            height: 100px;
            border:2px solid black;
            position: static;
        }
    
    </style>
</head>
<body>
    <div class="box">Box 1</div>
    <div class="box">Box 2</div>
    <div class="box">Box 3</div>
</body>
</html>

everything appears normally.
nothing changes.

Important point:
----------------
for static elements:
top
left
right
bottom
do not work.

Position 2 - Relative:
-------------------
moves an element relative to its original position.

------------------------------------------------
The CSS position property specifies how an html should be positioned on a webpage.

The position property tells the browser:
where should this element be placed, and what should it use as its reference.

Syntax:
------------
selector{
position:value;
}
 eg:
-------
.box {
position:relative;
}

Position: Relative
----------------
An element with position:Relative is positioned relative to its original location.
The element first occupies its normal position.
Then we can move it from that original position using offset properties.

Syntax:
---------
.box{
position:relative;
top:20px;
right:30px;
}    

 <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        body{
            font-family: Arial, Helvetica, sans-serif;
            background-color: #f4f6f8;
            padding:30px;
        }
        h1{
            text-align: center;
            margin-bottom: 30px;
        }
        .container{
            width:500px;

            padding:25px;
            border:3px solid black;
            background-color: white;
        }
         .box{
            width:250px;
            padding:25px;

            border:2px solid black;
            text-align: center;
            font-size:20px;

        } 
        .box1{

        }
        .box2{
            background-color: lightblue;

            top:40px;

            background-color: lightgreen;
        }

</head>
<body>

    <div class="box box1">Box One</div>
    <div class="box box2">Box Two</div>
    <div class="box box3">Box Three</div>
    </div>
</body>
</html>

Characteristics:
-------------------
A relative positioned element:
->Remains in the normal document flow
->keeps its original space reserved
->can be moved from its original space
->does not cause nearby elements to occupy its old space
->can become the reference parent for absolute child.

Position:Absolute
------------------
An element with position: absolute is removed from the normal document flow and placed relative to its nearest positioned ancestor.

Simple -> The element leaves its normal place.
Other elements behave as if it is not present.

What is a Positioned Ancestor?
----------------------------
  A positioned ancestor is a parent element whose position is not static.

parent position:relative;
child position:absolute;

Characteristics:
------------------
->is removed from the normal document flow
->does not keep its original space
->can overlap other elements
->uses top, right, bottom, and left
->searches for the nearest positioned parent
->uses the page or containing block when no positioned parent exists.

Without Positioned Parent:
----------------------------

Position:Fixed
---------------------
An element with Position: fixed is positioned relative to the browser viewport.
It remains in the same visible location even when the user scrolls.
Characteristics:
----------------------
->Is removed from the normal document flow 
->Uses the browser viewport as the reference
->remains visible during scrolling
->can overlap webpage content
->commonly offset Properties
